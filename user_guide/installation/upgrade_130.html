Actualizando desde 1.2 a 1.3

Nota: Las instrucciones en esta página asumen que está corriendo la versión 1.2. Si no ha actualizado todavía a esa versión por favor haga esto primero.

Antes de realizar una actualización debería tomar su sitio fuera de liínea reemplazando el archivo index.php con uno estático.

Paso 1: Actualice sus archivos de CodeIgniter

Reemplace los siguientes directorios en su carpeta "system" con la nueva versión:

Nota: Si ha hecho un desarrollo en los archivos especial en estas carpeta, por favor haga copias de ellos primero.

  • application/models/   (nuevo para 1.3)
  • codeigniter   (nuevo for 1.3)
  • drivers
  • helpers
  • init
  • language
  • libraries
  • plugins
  • scaffolding

Paso 2: Actualice sus archivos de error

La versión 1.3 contiene dos nuevas plantillas de errores ubicadas en application/errors, y por consistencia las otras plantillas han sido renombradas.

Si no ha retocado alguno de estos archivos, simplemente reemplace esta carpeta:

  • application/errors/

Si usted ha retocado sus plantillas de error, renombrelo de la siguiente manera:

  • 404.php   =  error_404.php
  • error.php   =  error_general.php
  • error_db.php   (nuevo)
  • error_php.php   (nuevo)

Paso 3: Actualice su archivo index.php

Por favor abra su archivo index.php principal (ubicado en su raíz). Debajo de todo el archivo cambie esto:

require_once BASEPATH.'libraries/Front_controller'.EXT;

Por esto:

require_once BASEPATH.'codeigniter/CodeIgniter'.EXT;

Paso 4: Actualice su archivo config.php

Abra su archivo application/config/config.php y agregue estos nuevos items:

/*
|------------------------------------------------
| URL suffix
|------------------------------------------------
|
| This option allows you to add a suffix to all URLs.
| For example, if a URL is this:
|
| www.your-site.com/index.php/products/view/shoes
|
| You can optionally add a suffix, like ".html",
| making the page appear to be of a certain type:
|
| www.your-site.com/index.php/products/view/shoes.html
|
*/
$config['url_suffix'] = "";
/*
|------------------------------------------------
| Enable Query Strings
|------------------------------------------------
|
| By default CodeIgniter uses search-engine and
| human-friendly segment based URLs:
|
| www.your-site.com/who/what/where/
|
| You can optionally enable standard query string
| based URLs:
|
| www.your-site.com?who=me&what=something&where=here
|
| Options are: TRUE or FALSE (boolean)
|
| The two other items let you set the query string "words"
| that will invoke your controllers and functions:
| www.your-site.com/index.php?c=controller&m=function
|
*/
$config['enable_query_strings'] = FALSE;
$config['controller_trigger'] = 'c';
$config['function_trigger'] = 'm';

Paso 5: Actualice su archivo database.php

Abra su archivo application/config/database.php y agregue estos nuevos items:

$db['default']['dbprefix'] = "";
$db['default']['active_r'] = TRUE;

Paso 6: Actualice su guía de usuario

Por favor también reemplace sus copias locales de la guía del usuario con la nueva versión.
